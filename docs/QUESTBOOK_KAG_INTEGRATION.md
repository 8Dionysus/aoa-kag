# QUESTBOOK integration — aoa-kag

## Purpose

This note shows how `QUESTBOOK.md` fits into `aoa-kag` as the public tracked surface for deferred derived-layer obligations.

## Role split

- source repos remain the owners of meaning
- `aoa-kag` remains the owner of derived, provenance-aware structures and bounded export contracts
- `QUESTBOOK.md` tracks obligations that survive the current diff at the derived layer
- quests here should point to bridge, exposure, provenance, readiness, or regrounding seams

## Good anchors in this repo

Use stable anchors such as:
- `CHARTER.md`
- `docs/KAG_MODEL.md`
- `docs/SOURCE_OWNED_EXPORT_DEPENDENCIES.md`
- `docs/FEDERATION_KAG_READINESS.md`
- `docs/BRIDGE_CONTRACTS.md`
- `docs/RECURRENCE_REGROUNDING.md`
- `docs/CROSS_SOURCE_NODE_PROJECTION.md`

## Initial posture

Good KAG quests normally answer:
- which derived export seam needs bounded repair
- which source-owned dependency needs to be made explicit
- which bridge or recurrence return rule is under-specified
- which projection pilot is still experimental and should not become primary truth

If a readiness-only donor export appears before the live federation spine widens,
reanchor the dependency quest instead of silently promoting that donor into
production coverage.

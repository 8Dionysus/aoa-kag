# VIA_NEGATIVA_CHECKLIST

This checklist is for `aoa-kag` as the derived knowledge substrate and
retrieval-ready structures.

## Keep intact

- packs with strong provenance and real retrieval value
- projection health signals and explicit quarantine posture
- regrounding seams back to sources

## Merge, move, suppress, quarantine, deprecate, or remove when found

- packs that mostly copy source text
- stale or weakly sourced projections
- generated structures with no clear consumer or retrieval gain

## Questions before adding anything new

1. What value does this pack add beyond the source repo?
2. Can weak packs be quarantined instead of treated as live substrate?
3. Is there a clean source-first fallback if this pack disappears?

## Safe exceptions

- material aggregations that reduce retrieval cost without distorting meaning
- temporary packs supporting an active migration

## Exit condition

- KAG should stay useful as substrate, not seductive as shadow canon.

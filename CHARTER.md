# AoA KAG Charter

## Purpose

`aoa-kag` is the knowledge substrate layer of the AoA ecosystem.

Its purpose is to make derived knowledge-ready structures explicit as reusable graph-friendly and retrieval-friendly surfaces rather than leaving them implicit inside source repositories or locking them into one framework too early.

## Mission

This repository exists to support KAG-layer work such as:
- defining reusable lifted knowledge units
- preserving provenance-aware derived structures
- normalizing nodes and edges without replacing authored meaning
- clarifying framework-neutral graph and retrieval readiness
- making KAG surfaces reviewable across AoA and ToS

## What this repository owns

This repository owns KAG-layer truth about:
- lifted knowledge units
- normalized node and edge projections
- provenance-aware derived surfaces
- retrieval-ready section and chunk maps
- framework-neutral KAG registries
- KAG-layer schemas and validation rules

## What this repository does not own

This repository does not own the primary meaning of:
- reusable techniques
- bounded execution workflows
- bounded proof surfaces
- memory objects
- routing surfaces
- authored ToS texts
- infrastructure implementation

## Core principles

- source repositories keep authored meaning
- provenance should remain visible where possible
- KAG surfaces should stay derived and reviewable
- graph readiness should be bounded rather than theatrical
- framework adapters should remain downstream of the substrate
- the KAG layer should not quietly replace the sources that feed it

## Source-of-truth rule

Neighboring AoA repositories and ToS sources still own their own meaning.

Examples:
- `aoa-techniques` owns practice meaning
- `aoa-skills` owns execution meaning
- `aoa-evals` owns proof meaning
- `aoa-memo` owns memory meaning
- `Tree-of-Sophia` owns authored knowledge-architecture content
- `aoa-kag` owns derived knowledge substrate meaning

## Role in the federation

Treat `aoa-kag` as:
- the derived knowledge substrate layer of AoA
- the canonical home of normalized KAG surfaces and provenance-aware lifts
- the place where framework-neutral graph and retrieval readiness becomes explicit

## Long-term direction

If `aoa-kag` matures well, it should help AoA and ToS accumulate linked knowledge without collapsing the distinction between:
- what was authored
- what was lifted
- what can be traversed or retrieved
- what still remains deferred rather than falsely implied

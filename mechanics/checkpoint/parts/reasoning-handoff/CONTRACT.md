# Reasoning Handoff Contract

This part keeps runtime-to-KAG reasoning handoff weak enough to return.

It requires:

- handoff outputs to stay derived guide-to-source surfaces;
- `source_refs`, `axis_summary`, and `provenance_note` to remain required return
  fields;
- counterpart refs to stay contract or example refs while `AOA-K-0008` remains
  planned;
- KAG guardrail refs to include the reasoning handoff doc, schema, example,
  counterpart consumer contract, and counterpart federation exposure review;
- memory readiness, durable consequence, retention, recall, and live-ledger
  pressure to route back to `aoa-memo`;
- proof and verdict pressure to route back to `aoa-evals`.

It forbids:

- treating a handoff pack as durable owner state;
- treating missing external owner roots as proof that a local example is bad;
- promoting counterpart refs into generated `AOA-K-0008` payloads;
- using active legacy `seed`, `wave`, `stub`, or `landing` naming as handoff
  stage semantics.

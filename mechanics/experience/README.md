# Experience

This package maps the common AoA Experience mechanic into the KAG layer.

## Mechanic card

Status: mapped common-center mechanic; active part-local routes exist for
governance precedent, release patterns, and office/service patterns.

### Trigger

Use this package when KAG carries governance, service, office, release, or
pattern surfaces that came from experience-derived cycles.

### Local owns

`aoa-kag` owns derived pattern contracts, public examples, schemas, and
source-linked KAG representations of experience pressure.

### Stronger owner split

`Agents-of-Abyss` owns common experience mechanic law. Owner repositories own
adoption and lived operation. `aoa-evals` owns certification proof.
`aoa-playbooks` owns repeatable choreography.

### Inputs

Experience docs, schemas, examples, governance precedent candidates,
release/installation pattern candidates, office/service candidates, and
focused tests.

### Outputs

KAG governance precedent nodes, release/installation pattern candidates, and
office/service pattern candidate surfaces.

### Must not claim

Owner adoption, lived operation, release execution, proof certification,
runtime state, or forced local change.

### Next route

Start with `PARTS.md`. Use active part homes when changing governance
precedent, release/installation, post-release, office, or service pattern
candidate docs, schemas, examples, or focused tests.

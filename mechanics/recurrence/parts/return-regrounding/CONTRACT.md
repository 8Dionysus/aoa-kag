# Return Regrounding Contract

This part keeps derived recurrence as a return route, not a new owner.

It requires:

- return modes to keep the stable order expected by KAG consumers;
- stronger refs to point outside `aoa-kag` local generated/docs/examples
  surfaces;
- counterpart review or contract refs to remain supporting context, not
  stronger authority;
- `handoff_guardrail_reentry` and `owner_boundary_reentry` to route memory
  readiness pressure back to `aoa-memo`;
- `source_export_reentry` to stay about source-owned export capsules, not memo
  readiness or routing activation;
- bounded output contracts to forbid source replacement, routing ownership,
  memory truth ownership, canon authorship, and proof ownership.

It forbids:

- treating regrounding as routing policy;
- treating memo readiness as KAG-owned retention, scar, or live-ledger proof;
- promoting local generated packs into stronger refs;
- using active legacy `seed`, `wave`, `stub`, or `landing` naming as recurrence
  stage semantics.

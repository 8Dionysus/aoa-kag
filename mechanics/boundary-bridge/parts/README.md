# Boundary Bridge Parts

`mechanics/boundary-bridge/parts/` holds active KAG-local boundary-bridge
sub-mechanics.

Use this home for functioning bridge routes with local contracts and
validation. Source docs, manifests, schemas, examples, and generated read
models may stay in their root homes while part-local validation owns the
operation contract around them.

## Active parts

| Part | Route |
| --- | --- |
| `source-owned-export` | validates source-owned export dependency posture, memo donor registry-only activation, and owner-primary ingress boundaries |
| `tos-retrieval-axis` | validates ToS retrieval handle posture, source-first reentry, and standalone adjunct limits |
| `cross-source-projection` | validates one-hop cross-source projection, primary/supporting input split, and counterpart exclusion |
| `federation-spine` | validates source-owned export spine posture, artifact identity, ToS adjunct budget, and counterpart exclusion |

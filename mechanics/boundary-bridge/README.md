# Boundary Bridge

This package maps the common AoA Boundary Bridge mechanic into the KAG layer.

## Mechanic card

Status: mapped common-center mechanic; no active part directories yet.

### Trigger

Use this package when KAG connects source-owned exports, ToS route surfaces,
memo bridge faces, counterpart edges, or cross-source projections.

### Local owns

`aoa-kag` owns derived bridge payload shape, source refs, non-identity
boundaries, generated projection contracts, and consumer return routes.

### Stronger owner split

Source repositories own authored meaning. `Tree-of-Sophia` owns canon.
`aoa-memo` owns memory objects. `aoa-evals` owns proof. `aoa-routing` owns live
dispatch. Runtime owners own runtime state.

### Inputs

Source-owned exports, bridge examples, manifest inputs, generated federation
spine inputs, counterpart refs, retrieval-axis handles, and source policy docs.

### Outputs

Bounded bridge contracts, generated federation and cross-source projection
surfaces, counterpart consumer posture, and return routes to stronger owners.

### Must not claim

Identity, source replacement, ToS canon, memory truth, proof verdict, routing
authority, graph sovereignty, or owner acceptance.

### Next route

Start with `PARTS.md`. Promote a bridge part only when one bridge family needs
its own contract, validation route, and nearest artifact home.

# Growth Cycle

This package maps the common AoA Growth Cycle mechanic into the KAG layer.

## Mechanic card

Status: mapped common-center mechanic; active part route:
`parts/surface-growth-stop-rule/`.

### Trigger

Use this package when KAG maturity, owner wait states, proof gaps, pause rules,
or surface-growth progression are in scope.

### Local owns

`aoa-kag` owns local maturity-governance payloads, current stop-rule posture,
owner wait state representation, and generated parity for growth claims.

### Stronger owner split

`Agents-of-Abyss` owns common growth-cycle law. Owner repositories provide
stronger exports or acceptance. `aoa-evals` owns proof. `aoa-playbooks` owns
repeatable growth choreography.

### Inputs

Maturity governance manifest, owner wait states, proof expectations, roadmap
contour, generated maturity pack, and decision records.

### Outputs

Paused or allowed growth posture, owner wait-state visibility, proof gap notes,
and current KAG surface maturity signals.

### Must not claim

Universal progression score, owner acceptance, proof verdict, automatic new
surface creation, or hidden scheduler authority.

### Next route

Start with `PARTS.md`. Use `parts/surface-growth-stop-rule/` for the current
maturity stop-rule route. Create another part only when a distinct growth route
needs its own KAG-local contract beyond current maturity governance.

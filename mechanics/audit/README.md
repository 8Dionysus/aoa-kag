# Audit

This package maps the common AoA Audit mechanic into the KAG layer.

## Mechanic card

Status: mapped common-center mechanic; active part routes:
`parts/proof-expectation-refs/` and `parts/exposure-review/`.

### Trigger

Use this package when a KAG surface needs evidence visibility, proof
expectation refs, owner-route risk, or exposure review before growth.

### Local owns

`aoa-kag` owns derived-surface audit posture: source refs, proof expectation
links, exposure-review artifacts, and stop-lines that keep claims weaker than
proof.

### Stronger owner split

`Agents-of-Abyss` owns common audit doctrine. `aoa-evals` owns proof verdicts.
Source repositories own source evidence and remediation. Runtime owners own
live operational evidence.

### Inputs

Proof expectation docs, eval catalog refs, exposure review manifests, generated
maturity packs, source-owned export evidence, and KAG validators.

### Outputs

Audit-visible derived posture, owner-route risk notes, proof expectation refs,
and blocked or deferred growth signals.

### Must not claim

Proof verdict, finding closure, owner acceptance, source remediation, runtime
state, or activation authority.

### Next route

Start with `PARTS.md`. Use `parts/proof-expectation-refs/` for proof-ref
audit posture and `parts/exposure-review/` for counterpart exposure review.
Promote `owner-evidence-route` only when it has a distinct audit payload beyond
the source-owned export bridge route.

# AGENTS.md

## Applies to

This card applies to the canonical `aoa-kag/skills/` owner home.

## Role

This home owns only admitted repository-specific callable procedures. The
`aoa-kag` bundle teaches an agent how to use the read-only KAG access plane and
return to canonical owners; it does not own sibling meaning or MCP runtime.

## Boundaries

- Keep one `aoa-kag` front door. Keep retrieve, relate, and research as
  internal modes unless held-out manual work proves a split adds value.
- Keep discover, search, read, traverse, and explain as MCP operations rather
  than multiplying skill bundles around tool names.
- Keep the MCP five-tool contract explicit and read-only.
- Keep proof, memory, routing, role, playbook, statistics, runtime, technique,
  and authored source decisions with their owners.
- Keep raw prompts, traces, task-local DAGs, temporary rubrics, and trial notes
  in the session, not in this home.
- Do not require `aoa-techniques` at runtime.
- Expose the admitted bundle once through the OS user profile. Do not create a
  same-name `.agents/skills/aoa-kag` copy.

## Validation

Manual isolated, negative, held-out, coexistence, source-return, degraded-route,
full-contract, and live MCP tasks decide usefulness. The shared home-port check
proves only package, owner, manifest, digest, and duplicate-projection
invariants.

## Closeout

Report the manual modes exercised, owner sources returned, live MCP posture,
OS-profile status, prompt-visible status, and any session-only trial material
removed.
